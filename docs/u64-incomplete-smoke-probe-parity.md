# U64 Incomplete Smoke Probe Surface

Date: 2026-04-14

## Purpose

This note defines the exact host-side workload generated by `vivipi` for the incomplete smoke surface. It is the relevant reproducer for the Ultimate 64 firmware issue under investigation.

## Reproducer

```bash
./scripts/u64_connection_test.py --profile soak --mode incomplete --surface smoke -d 0 --duration-s 7200
```

Observed repository states:

- `vivipi`: `69f0125f`
- `1541ultimate`: `5d4a6173`

Observed result:

- The machine eventually enters an incorrect state with recurring short audio noise.
- The network services are still reachable when the audio symptom first becomes obvious.

## Effective Probe Shape

With this command, `ping` and `http` stay on their normal correct-mode probes. The incomplete smoke surface only changes `ftp` and `telnet`.

FTP:

- open the control connection
- read the `220` greeting
- send `QUIT`
- read the reply
- close the socket

Expected result detail:

- `ftp greeting ready`

Telnet:

- open the telnet socket
- perform one initial read
- strip telnet negotiation bytes
- classify the first visible response as `login failed`, `banner ready`, or `connected`
- close the socket

Not done on telnet:

- no keystroke injection
- no menu navigation
- no repeated reads until idle
- no session reuse

## Scope Boundary

This note does not cover the incomplete `read` or `readwrite` surfaces.

Those surfaces intentionally exercise deeper protocol behavior, including partial FTP data-channel operations and telnet-side input injection. They are not part of this reproducer.

## Engineering Consequence

This workload repeatedly exercises the FTP control path while avoiding FTP data-channel work and keeping telnet shallow.

If the defect reproduces under this command, the first firmware inspection target is the FTP control-connection lifetime path and any shared state touched by frequent connect and disconnect cycles.
